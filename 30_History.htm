BioAPI Reference Implementation: History

A (Very) Brief History of the BioAPI Reference Implementation

  • July 13-15, 2000

    First development workshop, hosted by Bioscrypt (formerly Mytec).

  • August 22-23, 2000

    Second development workshop, hosted by SAFLINK.

  • September 13, 2000

    Version 1.0beta1 of the reference implementation released.

  • December 20, 2000

    Updates for the Version 1.0beta2 of the reference implementation sent to the reference implementation mailing list.

  • January 16, 2001

    Version 1.0beta2 of the reference implementation released.

  • February 19, 2001

    Version 1.01 of the reference implementation released.

  • March 16, 2001

    Version 1.10 of the reference implementation released.

  • March ??, 2002

    Version 1.11 of the reference implementation released. Changes include:

    • merged in changes made to CDSA release 3.14 (mainly improved portability, some bug fixes)
    • fixed Response parameter in GUI callback functions to be a pointer
    • updated Perl scripts and input files to generate correct output
    • fixed bugs dealing with non-zero device IDs
    • fixed dynamic device insert/remove issue
    • fix to allow multiple devices to be registered under a single module
    • improved this HTML documentation, particularly building and installation
    • improved password BSP and sample app to be better examples of "the right way"

    Any application which uses the GUI callback functions will be incompatible between version 1.10 and 1.11. However, since the callbacks could not operate as intended in version 1.10, it is expected that this will have little impact. Other applications will be able to work identically with both versions.
